# 신뢰도 필터 보완 기록

본 문서는 음식점 리뷰의 광고성/저신뢰 리뷰를 판별하기 위해 사용한 신뢰도 필터의 보완 과정을 정리한 것이다. 초기 모델은 짧은 리뷰와 일반적인 칭찬 표현을 중심으로 광고성 리뷰를 탐지하는 방식이었으나, 오분류 사례 분석을 반복하면서 실제 방문 경험을 나타내는 구체적 단서를 반영하는 방향으로 개선하였다.

## 1. 초기 모델: 단순 fake score 기반 필터

초기 필터는 리뷰가 광고성 또는 저신뢰 리뷰일 가능성을 나타내는 `fake_score`를 계산하는 방식으로 설계하였다.

주요 기준은 다음과 같았다.

* 리뷰 길이가 매우 짧은 경우
* “맛있어요”, “좋아요”, “친절해요”처럼 일반적인 칭찬 표현만 포함된 경우
* 계정 리뷰 수가 매우 적은 경우
* 사진이 없거나 방문 인증 정보가 부족한 경우
* 이벤트, 협찬, 제공 등 광고성 표현이 포함된 경우

이 방식은 구현이 간단하고 해석이 쉽다는 장점이 있었지만, 실제 리뷰의 문맥을 충분히 반영하지 못하는 문제가 있었다. 특히 짧지만 실제 방문자가 작성한 리뷰와, 길고 구체적이지만 광고성 문체를 가진 리뷰를 구분하는 데 한계가 있었다.

## 2. 2차 보완: 구체성 feature 추가

초기 모델의 오분류 사례를 검토한 결과, 단순히 리뷰 길이나 긍정 표현만으로는 신뢰도를 판단하기 어렵다는 점을 확인하였다. 이에 따라 리뷰에 실제 방문 경험이 담겨 있는지를 판단하기 위한 구체성 feature를 추가하였다.

추가한 주요 feature는 다음과 같다.

* 메뉴명 포함 여부
* 숫자 표현 포함 여부
* 가격, 인원, 대기 시간 등 구체적 정보 포함 여부
* 주문, 방문, 기다림, 재방문 등 경험 표현 포함 여부
* 구체적인 부정 표현 포함 여부

이 단계에서는 “리뷰가 긍정적인가”보다 “리뷰가 실제 경험을 구체적으로 설명하는가”를 더 중요하게 보았다. 예를 들어 부정 표현이 포함된 리뷰라도, 구체적인 경험과 함께 제시되면 오히려 신뢰도 높은 리뷰로 판단할 수 있도록 하였다.

## 3. 3차 보완: 이벤트 리뷰와 일반 리뷰 비교

이후 리뷰 이벤트 또는 광고성 리뷰와 일반 리뷰의 차이를 비교하였다. 이벤트성 리뷰는 사진과 긍정 표현을 포함하는 경우가 많기 때문에, 사진 첨부 여부나 긍정 표현만으로는 신뢰도를 높게 평가하기 어렵다는 점을 확인하였다.

이 단계에서 확인한 특징은 다음과 같다.

* 이벤트성 리뷰는 일반적 칭찬 표현의 비율이 높았다.
* 구체적인 경험 설명 없이 긍정 표현만 반복되는 경우가 많았다.
* 사진이 첨부되어 있더라도 실제 경험성이 약한 리뷰가 존재하였다.
* 메뉴명이나 숫자 표현이 포함되어도 광고성 문체와 결합되는 경우가 있었다.

따라서 사진 첨부나 긍정 표현은 강한 신뢰도 가점이 아니라 보조적 feature로 처리하였다. 반면 구체적인 경험 표현, 재방문 표현, 부정적 경험 설명 등은 신뢰도 판단에서 더 중요한 feature로 반영하였다.

## 4. 4차 보완: fake score에서 trust score로 전환

초기 모델은 광고성 가능성을 높이는 요소를 중심으로 `fake_score`를 계산하였다. 그러나 오분류 분석 결과, 저신뢰 요인만 누적하는 방식보다 신뢰도를 높이는 단서와 낮추는 단서를 함께 반영하는 방식이 더 적절하다고 판단하였다.

이에 따라 최종 모델에서는 `trust_score`를 사용하였다.

신뢰도 증가 요인은 다음과 같다.

* 메뉴명 포함
* 숫자 표현 포함
* 구체적인 경험 표현 포함
* 재방문 표현 포함
* 구체적인 부정 표현 포함
* 방문 인증 정보 존재
* 일정 수준 이상의 계정 리뷰 수

신뢰도 감소 요인은 다음과 같다.

* 리뷰 길이가 매우 짧음
* 일반적인 칭찬 표현만 포함
* 구체적 정보 부족
* 이벤트 또는 광고성 표현 포함
* 계정 리뷰 수가 매우 적음

이 방식은 각 리뷰를 단순히 광고성 여부로만 판단하지 않고, 실제 방문 경험의 구체성과 정보량을 기준으로 평가한다는 점에서 프로젝트 목적에 더 적합하였다.

## 5. 5차 보완: threshold 조정 및 최종 필터 적용

최종 단계에서는 수동 라벨링 데이터를 기준으로 `trust_score`의 threshold를 조정하였다. threshold가 너무 낮으면 광고성 리뷰가 고신뢰 리뷰로 남을 수 있고, threshold가 너무 높으면 실제 일반 리뷰까지 과도하게 제거될 수 있으므로, Accuracy와 F1-score를 함께 확인하였다.

최종 trust score 모델은 수동 라벨링 데이터 기준으로 다음과 같은 성능을 보였다.

| 지표                                   |       값 |
| ------------------------------------ | ------: |
| Accuracy                             | 약 0.812 |
| F1-score for low-trust/fake reviews  | 약 0.814 |
| F1-score for high-trust/real reviews | 약 0.811 |
| Test Accuracy                        | 약 0.820 |

또한 이전 버전과 비교했을 때 오분류 수가 감소하였다. 4차 모델에서는 전체 오분류 수가 626건이었으나, 5차 trust score 모델에서는 471건으로 감소하였다. 이는 리뷰의 표면적 특징뿐 아니라 경험성, 구체성, 계정 특성 등을 함께 반영한 결과로 해석할 수 있다.

## 6. 주요 오분류 유형

최종 모델에서도 다음과 같은 오분류 유형은 남아 있었다.

### 6.1 짧지만 실제 경험인 리뷰

예를 들어 “맛있어요”, “또 갈게요”, “여기 괜찮음”처럼 짧은 리뷰는 실제 방문자가 작성했을 가능성이 있지만, 정보량이 적기 때문에 저신뢰 리뷰로 분류될 수 있다.

### 6.2 길고 구체적이지만 광고성 문체를 가진 리뷰

체험단이나 리뷰 이벤트 리뷰는 메뉴명, 사진, 구체적 표현을 포함하는 경우가 많다. 따라서 길고 자세하다는 이유만으로 고신뢰 리뷰로 분류하면 광고성 리뷰가 남을 수 있다.

### 6.3 일반적 칭찬 표현과 실제 경험 표현이 섞인 리뷰

일부 리뷰는 “맛있어요”와 같은 일반적 칭찬 표현을 포함하면서도 실제 방문 경험을 함께 담고 있다. 이 경우 feature 간 가중치에 따라 고신뢰 또는 저신뢰로 다르게 분류될 수 있다.

### 6.4 부정 표현의 해석 문제

부정 표현은 감성분석에서는 낮은 점수로 반영되지만, 신뢰도 필터에서는 실제 경험을 나타내는 단서가 될 수 있다. 따라서 부정 표현을 단순히 낮은 품질의 리뷰로 해석하지 않고, 구체적 경험성과 함께 판단해야 한다.

## 7. 최종 판단 기준

최종 신뢰도 필터는 다음 원칙에 따라 설계되었다.

1. 짧은 리뷰나 일반적인 칭찬 표현만으로는 신뢰도를 높게 평가하지 않는다.
2. 메뉴명, 숫자, 방문 경험, 재방문 표현 등 구체적 단서가 있는 리뷰를 높게 평가한다.
3. 부정 표현은 감성 점수에서는 감점 요인이지만, 신뢰도 판단에서는 실제 경험의 단서로 볼 수 있다.
4. 사진 첨부와 방문 인증은 보조적 feature로만 사용한다.
5. 계정 리뷰 수가 매우 적은 경우 신뢰도를 낮게 평가하되, 단독 기준으로 사용하지 않는다.
6. 최종 판정은 개별 feature 하나가 아니라 trust score의 합산 결과와 threshold를 기준으로 수행한다.

## 8. 향후 개선 방향

향후에는 규칙 기반 trust score를 머신러닝 모델과 결합할 수 있다. 현재 사용한 feature들은 해석 가능성이 높기 때문에, Logistic Regression, Random Forest, XGBoost 등의 입력 변수로 활용할 수 있다. 이를 통해 feature importance 또는 SHAP 분석을 수행하면 신뢰도 판단에 어떤 요인이 가장 크게 기여했는지 더 정량적으로 설명할 수 있다.

또한 플랫폼별 리뷰 작성 문화가 다르므로, 네이버와 카카오에 동일한 threshold를 적용하는 대신 플랫폼별 threshold를 따로 설정하는 방식도 검토할 수 있다. 마지막으로 짧지만 실제 경험인 리뷰를 과도하게 제거하지 않기 위해, 짧은 리뷰 안에서도 재방문 표현, 강한 감성 표현, 특정 메뉴명 등을 더 세밀하게 반영하는 보완이 필요하다.
